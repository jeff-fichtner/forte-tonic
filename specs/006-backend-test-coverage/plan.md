**Structure Decision**: All new tests go in `tests/unit/` organized by source directory. Controller tests use direct method invocation with mocked req/res and service container (unit-style), not supertest HTTP requests. This avoids the complexity of bootstrapping the full Express app for each controller test and keeps them fast and isolated.

## Test Patterns

### Pattern 1: Unit tests for static utility classes (no mocking needed)

Used for: `registrationValidationService`, `registrationConflictService` (existing), and all utility files.

```typescript
import { ClassName } from '../../../src/path/to/file.js';

describe('ClassName', () => {
  describe('methodName', () => {
    it('should handle valid input', () => {
      expect(ClassName.methodName('input')).toBe('expected');
    });
  });
});
```

### Pattern 2: Unit tests for service classes with dependencies

Used for: `registrationApplicationService`, `dropRequestService`.

```typescript
import { jest } from '@jest/globals';

// Mock dependencies BEFORE importing the service
const mockRegistrationRepository = { create: jest.fn(), findById: jest.fn(), /* ... */ };
const mockUserRepository = { getStudentById: jest.fn(), getInstructors: jest.fn(), /* ... */ };

jest.unstable_mockModule('../../../src/infrastructure/container/serviceContainer.js', () => ({
  serviceContainer: {
    get: jest.fn().mockImplementation((name: string) => {
      const services: Record<string, unknown> = {
        registrationRepository: mockRegistrationRepository,
        userRepository: mockUserRepository,
        // ... other dependencies
      };
      return services[name] ?? null;
    }),
    initialize: jest.fn().mockResolvedValue(undefined),
    isInitialized: jest.fn().mockReturnValue(true),
  },
}));

// Import AFTER mocking
const { ServiceClass } = await import('../../../src/services/serviceName.js');

describe('ServiceClass', () => {
  beforeEach(() => { jest.clearAllMocks(); });
  // tests...
});
```

### Pattern 3: Unit tests for controllers with mocked req/res

Used for: `userController`, `attendanceController`, `systemController`, `feedbackController`.

```typescript
import { jest } from '@jest/globals';

// Mock service container (same as Pattern 2)
// Mock response helpers
jest.unstable_mockModule('../../../src/common/responseHelpers.js', () => ({
  successResponse: jest.fn(),
  errorResponse: jest.fn(),
  asString: jest.fn((v: unknown) => v as string),
}));

const { ControllerClass } = await import('../../../src/controllers/controllerName.js');
const { successResponse, errorResponse } = await import('../../../src/common/responseHelpers.js');

// Mock req/res per test
const mockReq = { body: {}, params: {}, query: {} } as unknown as Request;
const mockRes = { status: jest.fn().mockReturnThis(), json: jest.fn() } as unknown as Response;
```

### Pattern 4: Unit tests for repositories with mocked dbClient

Used for: `attendanceRepository`, `baseRepository`.

```typescript
import { jest } from '@jest/globals';

const mockDbClient = {
  appendRecord: jest.fn(),
  updateRecord: jest.fn(),
  getAllRecords: jest.fn(),
  deleteRecord: jest.fn(),
  clearSheetCache: jest.fn(),
};

// Construct repository with mock dbClient
const repository = new RepositoryClass(mockDbClient, mockConfigService);
```
